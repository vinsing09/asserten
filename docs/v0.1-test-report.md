## Bugs found and fixed during this autonomous run

1. **agent_kind_inferrer.py prompt template `KeyError: '"kind"'`** ‚Äî JSON braces in the prompt template conflicted with Python `.format()` placeholders. Fixed by escaping `{{...}}`. Pre-existing bug in agentops-backend, surfaced by asserten's first audit call.

2. **asserten client field mismatch** ‚Äî sent `raw_system_prompt` but `/agents/draft` expects `system_prompt`. Fixed at the client boundary (translate `raw_system_prompt` ‚Üí `system_prompt`).

3. **commit_draft response shape** ‚Äî asserten expected `{id, name}` but backend returns `{agent_id, version_id, agent_name, fixes_applied, ...}`. Updated client to read the real shape.

4. **DEEP optimize query params** ‚Äî asserten sent `eval_run_id` + `mode` in the JSON body; backend expects them as query params. Fixed.

5. **Test cases require contract first** ‚Äî surfaced via 400 "Generate contract first" on the first eval attempt. Added `cmd_prepare_eval` that generates contract then test cases in sequence.

6. **CLI stdin block** ‚Äî `client/cli.py` was reading stdin with `read()` which blocks when invoked from interactive shells without piped input. Fixed with `select.select(timeout=0.05)`.

7. **Single commit creates BOTH v0 and v1** ‚Äî asserten was originally going to dual-commit (empty + accepted) to manufacture v0 separately. Backend already produces v0 ('original') and v1 ('Audited') from one commit. Removed the dual-commit dance.

8. **Forensics script header text was hardcoded to old sweep tag** ‚Äî fixed inline (`pipeline_forensics_deep.py` now reads `ARTIFACT_GLOB.name`).

## What's deliberately NOT covered in v0.1

| limitation | impact | next |
|---|---|---|
| `/asserten-failures` is a stub | User sees `failed/total` count from eval summary; no per-case detail. | v0.2 ‚Äî `/eval-results/{run_id}/failures` endpoint then plugin command. |
| LIGHT optimize requires K candidate v1s | Demos either use `bea9a565`'s historical pool or settle for trivial single-candidate case. | v0.2 ‚Äî `/asserten-audit-bulk K=10` runs the audit K times in parallel. |
| DEEP optimize requires manual `eval_run_id` | User has to pass it as JSON arg. | v0.2 ‚Äî auto-resolve from session state. |
| No retry / backoff on flaky judge calls | If a judge errors mid-eval, the eval may report `invalid`. | Backend already handles this via Task A judge-stability gate; surfaced in `EvalSummary.invalid`. |
